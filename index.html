System Administration Guide

This document covers basic system administration tasks for Debian-based Linux distributions. All commands assume root or sudo access.

Getting Started

Before proceeding, ensure your package lists are up to date. Failure to do so may result in dependency conflicts or installation of outdated software.

Links open in the same tab by default. See the Debian docs for further reference.

Prerequisites

  • Debian 11 or later
  • Root or sudo access
  • Network connectivity
    • Wired preferred
    • Wireless requires firmware
  • At least 2GB free disk space

Installation

Update the package index, then install the required packages:

sudo apt update
sudo apt upgrade -y
sudo apt install -y vim git curl wget htop tmux

Verify the installation completed without errors. Check the exit code with echo $? — a value of 0 indicates success.

Configuration

  1. Edit /etc/apt/sources.list to add contrib and non-free
  2. Run apt update again
  3. Install any additional firmware
  4. Reboot the system

User Management

Adding a User

useradd -m -s /bin/bash -G sudo newuser
passwd newuser

Removing a User

userdel -r olduser

The -r flag removes the home directory. Omit it to preserve user files.

Common Tasks

Disk Usage

df -h
du -sh /var/log/*

Running Processes

ps aux | grep nginx
kill -9 <PID>

Task Checklist

  • Configure SSH key auth
  • Disable root login
  • Set up ufw firewall
  • Configure fail2ban
  • Schedule log rotation
  • Set up automated backups

Reference

Key Bindings

In tmux: Ctrl+b then c creates a new window. Ctrl+b then d detaches the session.

Glossary

daemon
A background process not attached to a controlling terminal.
inode
A data structure storing file metadata excluding name and content.
PID
Process Identifier. Unique integer assigned to each running process.

Notes

Never run untrusted scripts as root. Always read scripts before executing them, especially when piping curl output directly to bash.
